Zeyo솔류션과 몰인몰 솔류션 매칭작업

안녕하세요 히어로 네이션 서양훈 입니다.

몰인몰 기능을 통합한 통합 이커머스 플랫폼 개발 이슈를 간략히 적어보았습니다. :)

우선 간단히 Zeyo에 대해 말씀드리겠습니다.

Zeyo는 의류 사이즈 서비스 비교 솔루션입니다.

실제 지금 운영하고 있고

https://zeyo.co.kr/

Zeyo 메인 페이지 입니다.

옷 구매자가 의류 구입시 손쉽게 사이즈 비교 통해 원하는 사이즈를 선택 할수 있도록 합니다.

CLIENT(옷 쇼핑몰 판매자) - ENDUSER(옷 구매자) 통해 옷 구매자는 사이즈 문제로 망설이지 않아도 되고 판매자는 반품 비율을 줄이고 매출 증가 효과가 있습니다.

샘플 사이트 입니다.

http://heronation.cafe24.com/product/%EB%B9%88%ED%8B%B0%EC%A7%80%EB%B2%A0%EC%9D%B4%EC%A7%81/59/category/1/display/2/

  • 빨간색 네모 버튼을 클릭해주세요 :)

이제 Zeyo는 옷 사이즈 비교 솔루션을 넘어서 몰인몰로 확장 할려고 합니다.

경쟁사는 지그재그, 에이블리등 있겠네요.

여기서 이슈가 발생되는데 몰인몰 개발 자체가 거대하고 많이 비용 + 시간이 필요하다보니 다른 솔루션(지금 현재 결정된것은 없지만 WISA, 그누보드등 있습니다.)을 독립형으로 구입해서

기존 우리 Zeyo 솔루션에 반영할려고 합니다.

Zeyo 솔류션 사용되고 있는 개발 언어 입니다.

  1. JAVA Spring boot(REST API) + Vue.js
  2. 인증 방식은 JWT + Oauth 2.0
  3. DB는 Maria DB 입니다.

ZEYO 몰인몰 개발 진행에 있어서 사용될 예정인 언어는

  • APP: android, IOS
  • 몰인몰 파트너 관리자 페이지: 정확히 결정 되지 않았지만 아마 기존 그누보드 소스를 이용해서 진행 계획(PHP) 할것 같습니다.
  • APP에 사용되는 서버와 클라이언트 통신 방식: REST API

반영시 대표적으로 발생되는 문제점 및 이슈를 설명하자면

  1. 기존 사용하고 있는 Zeyo 회원, 카테고리, 상품등 테이블하고 그누보드 + 영카트 테이블 매칭 작업
  2. 기존 그누보드에서 사용되는 세션 로그인 방식이 아닌 토큰 기반 인증 방식 변경 작업
  3. APP에서 사용되는 REST API 작업(기존 그누보드 소스를 REST API 전용 소스 변환 작업)


※ 잘 안보이시면 https://www.erdcloud.com/d/E3qshch6Wvy2o7K4S 접속 부탁드립니다.
그리고 이것은 그누보드 + 영카트 ERD 입니다. https://www.erdcloud.com/d/mtCwZPhavZD6ZSdtA

간단하게 ZEYO ERD를 표현 해보았습니다.

위 내용을 보충 할려고 하는데요

일단 기존 ZEYO DB를 이용해서 즉 현재 서비스 운영되어 있는 회원, 카테고리, 상품 DB를 BASE로 해서 개발 하기를 원합니다.

그누보드에서 사용되어져 있는 회원, 카테고리, 상품 등 DB를 가능한 기존 사용하고 있는 ZEYO DB에 접목시키고자 합니다.

다른 방향이 있겠지만 기존 그누보드 DB query문 소스 및 비지니스 로직 수정 하게 된다면 많은 이슈가 나올것 같습니다.

그외 다른 추가적인 여러 이슈 문제점이 있겠지만 일단 가장 크게 발생될만한 문제점을 말씀드렸습니다!

몰인몰 자체가 쉽게 개발되지 않겠지만 우리 회사 사정상 당장 수익을 창출 할려면 몰인몰 개발은 불가피 할것 같습니다.

혹시 이 글을 읽고 다른 좋은 방법이나 추가 이슈 문제점이 있다면 답변 부탁드리겠습니다 :)
